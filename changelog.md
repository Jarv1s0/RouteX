# 更新日志

## v4.3.0

### 新增

- 适配 mihomo `v1.19.29`，补充 ShadowQUIC、OpenVPN 新版协商与 TLS 密钥字段、证书校验名称、TLS 混淆选项及代理提供器 `override-expr` 配置支持。
- 自动构建版本在设置页展示 `AUTOBUILD` 标记、构建编号和对应提交信息，便于确认测试包来源。
- 新增正式版与自动构建独立更新通道；可在软件更新设置中切换通道，自动构建使用递增预发布版本和独立签名清单持续更新。

### 修复

- 修复订阅自动更新只保存开关和间隔、实际未创建定时任务的问题；现在应用启动后会恢复远程订阅调度，并在订阅编辑、关闭自动更新或更新失败后按最新配置继续安排任务。
- 升级 `adm-zip` 至 `0.6.0` 并移除冗余类型包，修复恶意 ZIP 声明超大解压尺寸可能导致进程内存耗尽的问题。
- 升级间接依赖 `serde_with` 至 `3.21.0`，修复序列化空映射条目时可能触发 panic 的问题。
- 修复悬浮窗缺少 UI 与国际化上下文导致部分组件无法正常工作的问题，并补充独立窗口所需的 Tauri 权限。
- 修复悬浮窗网速显示跳变和图标旋转档位突变的问题；现在网速按近期样本平滑显示，空闲时保持缓慢旋转，并随流量连续加速。

### 优化

- 精简应用数据目录：Windows 用户目录由 `%APPDATA%\routex.app` 迁移至 `%APPDATA%\routex`，内核与辅助工具统一存放到 `bin`，并压平运行目录和 Web UI 的重复层级；升级时自动迁移数据并在兼容任务更新后清理旧目录。
- 设置中的“打开配置文件目录”现在直接打开 `%APPDATA%\routex\config`，安装器、卸载器和进程清理逻辑同时兼容新旧目录。
- 合并同一轮订阅自动更新产生的状态通知和内核重启，多个订阅同时到期时最多重启一次内核，并避免退出过程中重新拉起内核。
- 移除任务栏、状态栏网速显示及外部 TrafficMonitor 集成，清理对应配置、资源下载、进程管理与 IPC 链路。
- Tauri 开发命令默认关闭文件监听，避免与前端开发服务器重复监听。
- 调整悬浮窗外观，移除系统窗口阴影，并统一更新入口的下载图标。
