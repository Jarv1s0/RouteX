# 更新日志

## v4.2.0

> 重要升级提示：如果你当前使用的是 `v4.1.x`，升级前请先备份数据，卸载旧版本后再安装 `v4.2.0`。本次调整了桌面端资源和运行目录结构，安装完成后需要重新拉取订阅；直接覆盖安装可能导致旧配置无法承接。

### 新增

- 适配 mihomo `v1.19.29`，补充 ShadowQUIC、OpenVPN 新版协商与 TLS 密钥字段、证书校验名称、TLS 混淆选项及代理提供器 `override-expr` 配置支持。
- 配置页面补充更多常用内核选项，日常调整可以尽量在界面里完成。
- DNS 设置增加更多高级选项，复杂网络环境下更少需要手动改文件。
- Windows 新增“禁用 GPU 加速”开关；如果界面渲染异常，可以在系统设置中开启后重启应用。
- 规则页支持按需下载地理数据，缺少数据时可以直接在应用内补齐。

### 修复

- 修复 mihomo `v1.19.28` 调整 `/proxies` 接口后，订阅提供器节点可能无法在代理组和原生托盘中显示的问题。
- 修正 `empty-fallback` 配置模板和提示，明确回退目标必须是代理节点，不能使用代理组。
- 升级 ECharts、`quick-xml` 和 `quinn-proto`，修复图表渲染、XML 解析及网络传输依赖的已知安全风险。
- 修复拓扑图在特殊节点名称下，部分连接线无法显示的问题。
- 修复地理数据缺失提示误报的问题；只有当前配置确实需要这些数据时才会提醒下载。
- 修复 WebDAV 备份和恢复会带回本机状态的问题；现在会清理敏感凭据和设备绑定的启动状态，避免恢复到另一台设备后影响内核启动。
- 修复配置或地理数据更新后没有及时重新加载的问题，减少设置已更新但实际运行仍是旧状态的情况。
- 修复远程规则查看器缺少复制入口、右键菜单异常和底部按钮布局重叠的问题。
- 修复部分规则预览类型识别不准的问题，减少预览内容和实际效果不一致的情况。
- 修复侧边栏规则数量在开关远程规则、本地规则或覆写配置后刷新不及时的问题。
- 修复规则数量偶尔短暂显示为 0 的问题。
- 修复重启内核、重启应用或启动阶段侧边栏规则数量跳到临时错误值的问题。
- 修复直连模式下代理侧边栏仍显示上一次代理组节点的问题，现在会直接展示直连状态。
- 修复订阅和覆写操作失败后界面仍像成功的问题；失败时不再清空输入或关闭编辑窗口。
- 修复订阅更新时“使用代理”和“检查格式”设置没有真正生效的问题。
- 修复模式切换和虚拟网卡开关在保存失败时仍继续执行的问题，减少界面和实际运行状态不一致的情况。
- 修复规则测试偶发拿不到活跃连接，导致匹配结果误判的问题。
- 修复 Windows 虚拟网卡模式下旧适配器清理不完整的问题。
- 修复 Windows 安装包和便携包在新运行目录结构下的图标与文件异常。

### 优化

- 减少主界面启动后的后台加载和连接监听，降低首次进入后的内存占用和刷新压力。
- 优化连接和流量页面的高频刷新表现，减少长时间使用时的界面抖动。
- 订阅、规则、代理、统计、设置等页面的操作入口统一到标题区或紧凑操作区，减少卡片内部的按钮占位。
- 更新器弹窗保留发布说明，下载进度移到底部操作区，避免标题栏遮挡和下载过程中内容跳动。
- 默认使用应用内置主题，移除冗余默认主题资源；悬浮窗图标旋转默认关闭，需要时仍可在外观设置中手动开启。
- 轻量模式和禁用 GPU 加速开关调整到更容易发现的位置，状态和提示更清晰。

### 重构

- 清理未接入界面的 IP 检查弹窗，并收紧 Rust 依赖审计对构建期公告的豁免范围。
- 统一桌面端 Rust 源码格式，并补齐 mihomo IPC 文件的 `rustfmt` 兼容性。
- 整理桌面端资源、运行目录和启动流程，降低升级和打包出错的概率。
- 拆分订阅、覆写、DNS、代理、连接和服务启动等页面与逻辑，后续维护更清楚。
- 改善内核状态同步，减少不同页面互相影响导致的异常状态。
- 梳理桌面端底层逻辑，为后续维护留出更清晰的边界。
