# 更新日志

## v2.2.5

### ✨ 新功能
- 订阅管理页合并覆写管理与代理集合视图，支持在同一页面内切换管理订阅、覆写和 provider 资源。
- 常规设置新增 DNS 与域名嗅探管理弹窗，接管状态改为统一的按钮入口，便于集中调整相关配置。

### ⚡ 优化
- 网络拓扑页改为按容器尺寸计算初始视口，切换页签时首帧即可进入正确布局，减少视图跳动与尺寸错位。
- 拓扑图布局计算改为“结构变化才重排”，连接流量刷新仅更新节点与边的数据，高频连接场景下更流畅。
- `framer-motion` 从基础 UI 分包中拆出，主路由切换与出站模式切换改为更轻量的过渡实现，降低主界面首屏负担。
- 日志监听从应用壳层收回到日志页按需启停，代理组监听改为引用计数托管，减少全局常驻监听带来的后台开销。
- 编辑器能力继续收敛，轻量文本编辑不再加载 Monaco，并将 PAC 类型声明改为按需加载，缩小编辑器链路体积。
- DNS 与域名嗅探页面改为复用统一编辑器组件，页面与设置弹窗共享同一套表单与保存逻辑。
- Windows 提权、自启和资源准备链路统一切换到 `routex-service` / `routex-run` 命名，并收口相关常量定义。
- Mihomo 更新与准备脚本改为按 release 资产自动匹配下载文件，降低不同平台构建命名变化带来的升级失败风险。
- `TrafficMonitor` 资源下载改为直接跟随官方仓库最新 release 资产，减少对第三方转发发布源的依赖。
- 资源准备脚本正式接入 `pnpm build` / `pnpm verify` 链路，新环境构建会自动拉取 `routex-service` 与 `routex-run` 运行时资源。
- Linux 打包与 AUR 模板、启动脚本、desktop 配置统一切换到 `routex` 命名，补齐剩余品牌迁移。

### 🛠 修复
- 修复从其他页签切换到“网络拓扑”页时，右侧拓扑内容先以溢出样式渲染再回弹到适配布局的问题。
- 修复网络拓扑页在页签切换过程中使用临时隐藏策略导致的短暂白屏问题，恢复为首帧直接显示正确视口。
- 修复侧边栏仍保留独立覆写入口导致管理入口重复的问题，旧 `/override` 路由现在会自动跳转到合并后的订阅管理页。
- 通过覆盖 `@xmldom/xmldom` 到 `0.8.12`，修复 `electron-builder` 传递依赖带来的 Dependabot 高危告警。
- 修复 Windows 下资源准备过程中目标文件被系统占用时直接中断的问题，锁定文件现在会跳过更新并继续完成其余资源下载。
- 修复 macOS `pkg` 安装后置脚本仍引用 `Sparkle.app` 旧包名的问题，统一改为 `RouteX.app`。

## v2.2.4

### ✨ 新功能
- 新增覆写回滚入口，出现异常时可以更快恢复并重新拉起内核。
- 工具页与订阅页加入远程图标本地缓存，减少重复下载与加载闪烁。
- 工具页“全部测试”加入并发上限控制，避免一次性打满检测请求。
- 自动构建版本统一追加 `-AutoBuild` 标识，方便识别构建来源。

### ⚡ 优化
- 主应用监听改为按页面启停，减少后台无效事件处理。
- SSID 检测改为按配置懒启动，并支持配置变更后的动态刷新。
- 网络健康监控收敛到主进程，补充 DNS 延迟采样，减少渲染层重复探测。
- 连接页过滤与排序链路增加缓存，高连接数场景下更流畅。
- 应用更新检查改为启动检查 + 回前台按时间窗补刷，降低固定轮询消耗。
- 订阅与覆写配置更新链路进一步收敛，减少重复写入和同步逻辑。
- 主进程入口职责拆分，启动、退出、深链和窗口生命周期更清晰。
- 连接快照改为主进程合并推送对象，减少高频遥测链路中的重复 JSON 解析与无效节流。
- 渲染层 IPC 拆分为领域模块，主进程 `system` 处理器与 shared 通道定义同步模块化，后续新增 IPC 的改动面更小。
- 事件收发统一收敛到 `onIpc/sendIpc + ON/SEND`，并删除 renderer 侧 `utils/ipc` 兼容层，彻底收口旧入口。
- ESLint 增加防回退限制，禁止重新导入旧 IPC 入口和直接访问 `ipcRenderer`。

### 🛠 修复
- 修复零参数 IPC handler 直接传入 `ipcErrorWrapper` 导致 event 对象泄漏到底层实现函数的问题。
- 修复 `isValidListenAddress` 中端口校验永远通过的 Bug（`!isValidPort()` 对对象永远为 false）。
- 修复 `processedConnIds` 仅在超过 2000 条时才清理，改为每次连接快照后即时回收已关闭连接 ID。
- 修复 `mihomoConnections` IPC 通道名硬编码绕过类型系统，统一改用 `IPC_ON_CHANNELS` 常量。
- 修复 `factory` 中链注入失败被静默丢弃，现改为输出警告日志便于排查。
- 替换 `generateProfile` 与 `injectChainProxies` 中三处 `JSON.parse/stringify` 深拷贝为 `structuredClone`。
- 删除混入 `utils/` 目录的手工测试文件，避免其被打包进产物。
- 修复 IPC 错误包装器在返回 `null` 时触发告警的问题。
- 修复统计页规则详情弹窗导致无限重渲染的问题。
- 修复 Windows 自定义无边框窗口下右上角按钮点击冲突的问题。
- 修复连接详情弹出卡片未跟随最新连接快照更新的问题，并将连接流量与速度刷新间隔统一收敛到 1 秒。
- 清理流量统计与网络 IPC 的类型和副作用问题，恢复 `typecheck` 与 `build` 验证链路。
- 升级 `yaml` 并锁定 `brace-expansion`、`picomatch` 安全版本，清理 Dependabot 依赖告警。
- 收紧 preload 暴露面并对白名单通道执行 `send/on/invoke` 访问校验，阻断未授权 IPC 通道调用。
- 修复 Windows 下切页后最小化并从托盘恢复时误触发整窗 reload，导致全屏白屏和主题样式闪动的问题。
