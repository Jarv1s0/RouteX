# 更新日志

## v3.2.0

### ⚠️ 注意

- 服务鉴权已切换到新的 `service-auth.json` 存储和 Auth V2 机制；旧版 `serviceAuthKey` 配置不再兼容，使用系统服务模式的用户升级后需要重新初始化服务。

### ✨ 新增

- 新增连接页快速规则：可从连接记录直接创建规则，并由独立的 `quick-rules.yaml` 管理，避免再生成临时覆写文件。
- 支持系统服务模式启动并管理 Mihomo 核心，服务模式下控制器请求会通过 `routex-service` 转发。

### ⚡ 优化

- 适配新版 `routex-service`：服务初始化会绑定当前用户 SID/UID，服务 API 请求升级为 Auth V2 签名。
- 适配 Mihomo `v1.19.24`，同步更新配置编辑器 schema，支持最新内核新增的配置项校验与补全。
- 新增内核新版本提醒：检测到 Mihomo 有新版本时，左侧「内核设置」版本号旁会显示小点提示。
- 优化合并订阅交互：点击未启用订阅会单独切换到该订阅；点击已合并订阅会将其设为主订阅，并把其他已合并订阅挂到新的主订阅下。
- 重构规则管理页面：将“本地规则”与“远程规则”分离显示；在远程规则页将规则条目与对应订阅管理整合至单一卡片，并严格按内核真实生效顺序排序展示；引入视觉引导线（Leader Line）极大地优化了规则页面的对齐阅读体验。
