# 更新日志

## v4.0.0

RouteX `4.0.0` 是项目基于Tauri的桌面版本。这个版本以 Tauri 2 作为桌面运行时，围绕 Mihomo 内核提供代理控制、订阅管理、规则编辑、连接观测、流量统计和常用网络工具，目标是让日常代理使用和复杂配置排查都能在一个桌面客户端里完成。

### ✨ 首版重点

- 发布 RouteX 首个正式桌面客户端，基于 Tauri 2 构建，覆盖 Windows、macOS 和 Linux。
- 内置 Mihomo 核心运行时，支持核心启动、停止、重启、配置生成、运行日志和控制器连接。
- 支持系统代理、TUN、DNS、Sniffer、外部控制器、运行环境变量和核心权限模式等常用 Mihomo 能力。
- 提供完整的订阅配置管理，包括本地配置、远程订阅、配置切换、排序、编辑、更新和状态展示。
- 支持覆写配置管理，可为订阅绑定覆写文件，并在客户端内编辑、启用和维护覆写内容。
- 提供规则管理与快速规则能力，支持查看规则、编辑规则、批量开关，并可从连接快速创建规则。
- 提供策略组与节点管理视图，支持节点选择、筛选、测速、链路查看和代理组状态展示。
- 提供实时连接页，支持活动连接与已关闭连接查看、筛选、排序、详情面板、连接关闭和拓扑视图。
- 提供统计中心，展示实时流量、历史流量、规则命中、Provider 使用情况、排行和运行状态。
- 提供日志页，可在客户端内查看 Mihomo 运行日志并打开详情。
- 提供网络工具页，包含 IP 信息、连通性检测、规则匹配、DNS 查询和流媒体可用性检测。
- 支持系统托盘、托盘菜单、浮动窗口、深链路、全局快捷键、窗口置顶和退出确认等桌面能力。


### 🛠️ 兼容性与稳定性

- 调整 Tauri 资源解析路径，确保安装版能读取内置核心、服务程序、geo 数据和图标资源。
- 区分开发版、自动构建版和正式发布版的 Windows 任务计划名称，避免不同构建互相覆盖启动任务。
- 对齐开发版与安装版的全局快捷键插件加载行为，避免安装版首次无快捷键配置时后续注册失败。
- 补齐管理员权限、服务模式、TUN、系统代理、托盘图标和窗口状态相关的桌面流程。
- 为渲染层加入生产环境 CSP 元信息，降低打包后页面资源加载差异。
- 增加 Rust 单元测试入口，并将 Rust tests 接入本地验证和 GitHub 构建流程。
- 接入 Tauri 官方 updater 插件，发布包生成签名更新产物，并通过 `latest.json` 为后续版本提供应用内更新。

### ⚠️ 已知说明

- 首版重点覆盖核心代理、配置管理和桌面体验，跨平台功能以各系统实际权限模型和 Tauri bundler 产物为准。
- Windows 下涉及 TUN、系统代理、服务模式和任务计划的功能可能需要管理员权限或系统安全策略允许。
- Windows 用户升级或重新安装后，必须先删除任务计划程序中旧版本残留的 RouteX 任务，再重新注册任务计划；可能需要清理的任务名包括 `routex`、`routex-run`、`routex-dev`、`routex-run-dev`、`routex-autobuild` 和 `routex-run-autobuild`。
- Windows 首次安装或清理旧任务后，必须右键 RouteX 选择“以管理员身份运行”启动一次，用于重新注册提权任务；注册完成后，后续可正常双击启动。
