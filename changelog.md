# 更新日志

## v4.2.0

> 重要升级提示：如果你当前使用的是 `v4.1.x`，升级前请先备份数据，卸载旧版本后再安装 `v4.2.0`。本次调整了桌面端资源和运行目录结构，安装完成后需要重新拉取订阅；直接覆盖安装可能导致旧配置无法承接。

### 新增

- Mihomo 配置模板补充新版内核相关选项，更多能力可以直接在界面中配置。
- DNS 高级设置支持更多解析方式，复杂 DNS 配置不再需要频繁手动编辑文件。
- Windows 新增“禁用 GPU 加速”开关；遇到界面渲染或悬浮窗稳定性问题时，可在系统设置中开启并重启应用生效。
- 规则页新增 GeoData 按需下载能力，缺少地理数据时可以直接在应用内补齐。

### 修复

- 修复部分依赖的安全风险。
- 修复拓扑图在特殊节点名称下，部分连接线无法显示的问题。
- 修复 GeoData 缺失提示误报的问题；只有当前配置确实依赖地理数据时，才提示前往 GeoData 页面下载。
- 修复 WebDAV 备份和恢复会带回本机状态的问题；现在会清理敏感凭据和设备绑定的启动状态，避免恢复到另一台设备后影响内核启动。
- 修复运行配置或 GeoData 更新后 Mihomo 内核没有及时重启的问题，减少配置已经更新但运行时仍使用旧状态的情况。
- 修复 Windows TUN 模式下遗留适配器清理不完整的问题。
- 修复 Windows 安装包和便携包在新运行目录结构下的图标与文件异常。

### 优化

- 减少主界面启动后的后台预加载和连接状态监听，降低首次进入后的内存占用和刷新压力。
- 优化连接和流量页面的高频刷新表现，减少长时间使用时的界面抖动。
- 订阅、规则、代理、统计、设置等页面的操作入口统一到标题区或紧凑操作区，减少卡片内部的按钮占位。
- 更新器弹窗保留发布说明，下载进度移到底部操作区，避免标题栏遮挡和下载过程中内容跳动。
- 默认使用应用内置主题，移除冗余默认主题资源；悬浮窗图标旋转默认关闭，需要时仍可在外观设置中手动开启。
- 轻量模式和禁用 GPU 加速开关调整到更容易发现的位置，设置项状态和提示更清晰。

### 重构

- 重组桌面端资源、运行目录和启动流程，并增加发布前检查，降低升级和打包出错的概率。
- 拆分订阅管理页面，降低单个页面的体积，让订阅、订阅组和覆写配置的维护边界更清楚。
- 拆分配置层 IPC 处理逻辑，按应用配置、链式代理、订阅、覆写和快捷规则划分文件，减少后续增删接口时的冲突。
- 拆分 DNS 设置编辑器，把表单状态、校验和保存逻辑移到独立 Hook，让页面组件只负责展示。
- 拆分核心进程管理逻辑，将配置规范化、Mihomo API 请求、就绪检查、启动错误诊断和生命周期流程分开维护。
- 拆分代理面板的测速、自动测速和列表视图模型逻辑，保留原有测速 TTL、并发控制和虚拟列表渲染行为。
- 改善 Mihomo 状态同步，减少页面之间互相影响导致的异常状态。
- 拆分桌面端订阅配置、运行目录、日志轮转和 GeoData 检测逻辑，降低后续维护和排查成本。
- 进一步梳理桌面端底层逻辑，为后续维护留出更清晰的边界。
