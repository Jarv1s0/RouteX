# 更新日志

## v4.0.0

RouteX `4.0.0` 是桌面端重写后的第一个正式版本。本版本迁移至 Tauri 2，并重新整理 Mihomo 启停、配置管理、规则维护、连接查看、流量统计和网络工具等核心功能；同时完成安装包、系统托盘、全局快捷键、深链路和应用内更新等桌面端能力的接入。

### 版本定位

- 本版本是 RouteX 桌面端重写后的首个正式版本，不属于旧版本的维护更新。
- 桌面运行时改为 Tauri 2，窗口、资源加载、权限、托盘、快捷键、深链路和更新流程均已重新接入。
- 构建目标包括 Windows、macOS 和 Linux，具体安装包格式以本次 Release assets 为准。
- 本版本优先保证核心流程可用：安装、启动、运行 Mihomo、维护配置、排查规则和查看连接。

### 主要变化

- Mihomo 管理：内置核心和配套资源，支持启动、停止、重启、日志查看、控制器连接、系统代理、TUN、DNS、Sniffer、环境变量和权限模式。
- 配置管理：支持本地配置、远程订阅、配置切换、排序、编辑、更新、状态展示、覆写配置绑定和 Proxy Provider 管理。
- 规则与策略：支持规则、规则 Provider、快速规则、从连接创建规则、代理组、节点筛选、节点测速、链路查看和代理组状态查看。
- 连接与日志：支持活动连接、已关闭连接、连接详情、连接关闭、拓扑视图和 Mihomo 运行日志。
- 统计与工具：支持实时流量、历史流量、账期统计、规则命中、Provider 使用情况、流量排行、IP 信息、连通性检测、规则匹配、DNS 查询和流媒体检测。
- 桌面端：支持系统托盘、托盘菜单、浮动窗口、深链路协议、全局快捷键、窗口置顶、退出确认和跨平台安装包。

### 打包与稳定性

- 修正安装版资源路径，安装后可以正常读取内置核心、服务程序、geo 数据和图标资源。
- 区分开发版、自动构建版和正式发布版的 Windows 任务计划名称，避免不同构建互相覆盖。
- 修正安装版全局快捷键加载流程，避免首次没有快捷键配置时后续注册失败。
- 补齐管理员权限、服务模式、TUN、系统代理、托盘图标和窗口状态相关的桌面流程。
- 为生产环境页面加入 CSP 元信息，减少打包后资源加载差异。
- 将 Rust 单元测试接入本地验证与 GitHub 构建流程。
- 接入 Tauri 官方 updater，发布包生成签名更新文件，并通过 `latest.json` 支持后续应用内更新。

### 安装与升级说明

- Windows 下涉及 TUN、系统代理、服务模式和任务计划的功能需要管理员权限，也可能受系统安全策略影响。
- Windows 用户升级或重新安装后，如遇到提权、TUN 或系统代理相关功能异常，请先删除任务计划程序中的旧 RouteX 任务，再以管理员身份启动 RouteX 重新注册。
- 可能需要清理的旧任务名包括 `routex`、`routex-run`、`routex-dev`、`routex-run-dev`、`routex-autobuild` 和 `routex-run-autobuild`。
- 清理旧任务或首次安装后，请右键 RouteX 选择“以管理员身份运行”启动一次；提权任务注册完成后，后续可正常双击启动。
