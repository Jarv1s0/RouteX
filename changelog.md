# 更新日志

> 重要升级提示：如果你当前使用的是 `v4.1.x`，升级到 `v4.2.0` 前必须先备份旧数据，然后完全卸载旧版本并重新安装；安装完成后需要重新拉取订阅。当前版本未保留 `4.1` 数据目录和旧快速规则的自动迁移逻辑，直接覆盖升级存在配置无法自动承接的风险。

## v4.2.0

### 新增

- Mihomo 配置模板补充 v1.19.27 相关配置辅助项。
- 新增 IPC 合同检查脚本，并接入 `verify` 校验流程，提前发现 Rust 与 TypeScript 通道声明不一致的问题。

### 修复

- 修复 React Router 相关依赖安全风险。
- 修复拓扑图节点名称包含 `|` 时部分连接线不显示的问题。
- 修复 GeoData 错误识别过宽，避免 geosite 分类不存在时误提示下载 GeoData 文件。
- 修复 WebDAV 备份包包含 `webdavPassword` 的问题，导出时会自动移除该敏感字段。
- 修复 Mihomo HTTP 请求可能被系统代理干扰的问题，请求现在显式绕过代理。

### 优化

- 优化订阅和规则管理卡片操作区布局，使按钮更紧凑。
- 默认使用应用内置主题，移除冗余默认主题资源。
- 重构桌面端资源和运行时目录结构，统一 `core`、`tools`、`config`、`runtime` 目录布局。
- 优化 Windows 提权启动任务、运行时日志目录和图标资源加载逻辑，减少安装路径变更后的兼容问题。
