# 更新日志

## v4.1.0

> **⚠️ 重点提示：GeoData 数据库文件不再在安装包内预置**
>
> - 如果您的配置依赖了 GEO 分流规则，应用在启动内核时会拦截并抛出精准提示，不会引发闪退。
> - 当遇到此提示时，您只需前往**「规则 - GeoData」**页面，按需一键下载相应数据库文件即可正常使用。

### 修复

- 修复了“订阅管理”页面顶部工具栏偶尔会变成两行显示的排版问题，现在各种按钮和输入框能完美对齐在同一行。
- 修复了在部分自定义主题下，“订阅管理”和“规则”页面顶部卡片背景效果不一致的问题，统一了软件的视觉体验。
- 修正删除当前参与运行配置的订阅后，运行时配置没有立即刷新，导致全局覆写或合并配置未按当前订阅组合重新生效的问题。
- 修正本地规则引用的代理组在当前运行配置中不存在时仍被注入的问题；现在会固定跳过这类规则，避免生成无效运行时规则。

### 优化

- DNS 设置页补齐 `listen`、`ipv6-timeout`、`prefer-h3`、`fallback`、`fallback-filter`、`direct-nameserver-follow-policy` 和 `cache-algorithm` 等配置项，并保留运行时 Fallback DNS 配置。
- 本地规则页会标记当前运行配置中不存在的代理组，显示“代理组不存在”，并在提示中说明运行时会跳过该规则。
- 本地规则注入改为在订阅、覆写、受控配置和代理链处理完成后执行，确保代理组存在性校验基于最终运行配置。
- 网卡接口查询逻辑从 PowerShell 脚本调用重构为 Rust 原生实现 (`get_if_addrs`)，大幅降低进程开销并提供了跨平台支持。
- 高频 IPC 调用路由逻辑从 O(n) 的字符串遍历分发重构为基于 `HashMap` 与 `OnceLock` 的 O(1) 原生底层调度，彻底解决高频率拉取状态时的耗时瓶颈。
- WebDAV 界面布局移除冗余注释并调整行高与窗口宽度，提供更紧凑精美的视觉体验。
- 清理近期重构产生的大量 Rust 未使用导入警告，提升代码整洁度与编译产物质量。
- 系统资源监控彻底移除了外部命令行依赖，重构为 Rust 底层原生接口，性能与跨平台稳定性大幅跃升。
- 系统剪贴板交互逻辑移除了外部脚本调用，全平台统一重写为原生内存级操作。
- 内核资源下载与解压不再依赖系统内置命令，转为使用原生库进行内存流式处理。
- 外部链接和本地目录的唤醒逻辑改为使用原生安全的库处理，避免了命令拼接的风险。
- 文件对话框移除了针对特定系统的 Shell 脚本，统一重构为全局跨平台原生实现。
- 精简了底层的加密与哈希依赖模块，进一步减小了软件安装包体积。
- 优化了节点测速与网络诊断逻辑，支持全并发探测，极大降低了极限网络探测与测速耗时。
- 优化了海量代理节点解析的内存分配机制，显著缓解了数万级别节点订阅场景下的瞬间卡顿与内存激增问题。
- 精简了软件安装包体积，优化了各类界面的过渡动画逻辑，使整体交互更加轻量流畅。
- 优化了 GeoData 配置页面的界面布局与下载反馈体验，排版更加美观紧凑，并采用了更友好的应用内消息代替系统通知。
- 统一日志、订阅和规则等页面顶部工具栏的玻璃样式实现，移除旧主题专用样式入口，减少自定义主题下的视觉偏差。
- 优化连接列表节点标签的视觉层级，使节点名称在浅色和深色主题下更清晰、更克制。
- 完善死代码检查脚本，支持识别 Vite `?worker` 导入，并正确处理仅在 package scripts 中使用的工具依赖。

### 重构

- 拆分并整理了“订阅管理”界面的代码结构，去除了臃肿的逻辑，提升了该页面的加载速度和未来的可维护性。
- 拆分 `mihomo-ipc` 中的运行时配置缓存与 WebSocket 事件桥接逻辑，清理旧 socket bridge 实现，降低实时状态同步模块的维护复杂度。
- 重构连接页面的状态管理，移除冗余的 `useConnectionsPage` Hook 与跨组件的 Ref 传递，将工具栏直接合并进 `ConnectionsListContainer` 以精简组件树与数据流动。
- 精简统计页及网络控制的代码层级，展平冗余控制流并移除无效的状态传递。
- 桌面端底层代码深度清理了冗余的分支控制与宏限制，代码更为精简整洁。
- 拆分重构了后端基础工具库与持久化存储模块，大幅提升底层架构的稳定性与可扩展性。
- 深度清理并优化了内核进程管理的底层调度与生命周期逻辑，内核启动与状态维护更为健壮。
- 统一并规范了后端全局的异常处理机制，使环境或配置异常导致的错误提示更为精准和标准化。
