# 更新日志

## v2.2.4
- 在覆写卡片菜单中提供回滚入口，便于覆写异常时快速恢复并重新拉起内核。
- 优化代理链管理弹窗样式，收敛表单布局与字号层级，提升编辑态与列表态的可读性。
- 自动构建版本改为通过环境变量注入 `AutoBuild` 标识，应用设置与菜单版本展示统一追加 `-AutoBuild`，避免构建流程直接改写源码与配置。
- 优化订阅与覆写配置更新链路，收敛重复的配置写入与同步逻辑，减少维护成本。
- 清理网络 IPC 中的 `any` 类型告警，并为连接/流量相关 store 补齐 `throttle.cancel()` 清理，降低页面切换时的残留副作用风险。
- 拆分主进程入口职责，抽离提权启动、退出流程、深链导入、窗口生命周期与启动编排逻辑，提升主进程可维护性并降低后续改动风险。

- 调整统计页流量汇总与排行榜的配色和图表表现，统一上传/下载主色，并收敛排行榜进度条与编号视觉。
- 优化全站侧边栏与表单状态图标，去除 fa、cg、io 等杂乱图标库，深度重构为统一的 Lucide 的图标集，并重置操作按钮的尺寸到标准的 18px 以保持界面细节的细腻一致。
- 重构统计页“订阅数据”折线图卡的色彩逻辑，弃用原先固定的硬编码刺眼蓝紫配色，全面接入系统级随动主题色 (`primary` 和 `warning` 变量)，大幅提升在各主题下的视觉护眼度和仪表盘专业感。

## 历史记录

### ⚡️ 性能优化
- 主窗口恢复逻辑减负，移除主进程中基于 `executeJavaScript` 的白屏探测，改为基于加载状态和崩溃状态的轻量恢复判断。
- SubStore 前端首页缓存，服务启动时读取并注入一次 `index.html`，避免每次请求重复读盘和字符串替换。
- 依赖与构建链路收敛，构建脚本统一切到 `pnpm`，移除仅单点使用的 `date-fns`。
- 连接页资源预加载瘦身，改为仅按可视区域和缓冲区预加载图标与应用名。
- 地理定位缓存下沉到主进程，增加超时、缓存和并发去重。
- 去掉拓扑页无实际使用的代理数据拉取，减少无效刷新和额外重排。
- 去掉渲染层重复启动 TrafficMonitor 的逻辑，避免重复拉起和状态抖动。

### 🛠 稳定性修复
- 托盘和悬浮窗相关 `ipcMain` 监听改为模块级单次注册，避免窗口反复开关后监听器累积。
- 悬浮窗和自定义托盘菜单改为只移除自身注册的 `mihomoTraffic` 监听，避免误删其他页面订阅。
- 去掉未实际消费的网络健康监控默认启动，减少常驻后台任务和无效事件推送。
- 应用退出前统一等待流量统计落盘、订阅统计落盘和系统代理回滚完成。
- `providerStats` 的快照、清空和保存改为串行队列执行，避免定时任务和手动操作互相覆盖。
- 修复覆写页拖拽上传事件未正确解绑的问题，避免重复监听和状态残留。
- 统一前端错误提示入口，减少页面间交互割裂和重复处理逻辑。

### 🎨 UI 与交互
- 抽离统一的弹窗样式与右上角关闭按钮，统一二级弹窗头部和关闭交互。
- 清理散落的 `alert()`，统一接入 Sonner 和全局对话框。
- 收窄全局 focus 样式作用范围，避免覆盖日志复制等行为。

### 🔐 安全修复
- 升级并覆盖 `tar`、`flatted`、`yauzl` 相关依赖，清除当前审计中的高危与中危告警。
