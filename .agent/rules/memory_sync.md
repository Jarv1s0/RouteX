# .ai_memory 主动同步协议 (Memory Sync Protocol)

> **适用对象**：所有接入本项目的 AI Agent
> **核心目标**：确保项目背景、任务状态和开发日志始终与实际进度同步，消除手动维护成本。

## 1. 触发条件 (Triggers)
在满足以下任一条件时，必须执行本协议：
- **任务交付前**：在准备调用 `notify_user` 告知用户任务完成前。
- **重大变更后**：完成核心架构修改、新增关键依赖或修复复杂 Bug 后。
- **会话结束前**：如果预计当前对话即将结束。

## 2. 操作规范 (Operations)

### 2.1 更新项目上下文 (`.ai_memory/1_project_context.md`)
- 若任务涉及技术栈变更（升级版本、新增库）、架构调整或核心业务逻辑改变，必须同步更新此文件。
- 确保“核心共识”部分始终准确。

### 2.2 更新开发流水账 (`.ai_memory/3_work_log.md`)
- **强制性要求**：每次任务结束必须追加一条记录。
- **格式**：使用 `## YYYY-MM-DD` 标题。
- **内容**：简明扼要地列出本次提交/修改的核心要点。

## 3. 质量控制 (Quality Control)
- **严禁虚假承诺**：只记录已经通过验证或确实执行的操作。
- **保持简洁**：使用列表形式，避免冗长的叙述。
- **中文优先**：日志内容必须使用中文编写，确保用户易读。
- **排除项**：严禁在此目录下存放 `implementation_plan.md` 或其他临时性技术文档，此类文档应通过 AI 的 Artifacts 功能管理。
